# Habit Tracker — выгрузка по проекту

Минималистичный трекер привычек в чёрно‑белом стиле, с лёгкой геймификацией
(опыт/уровни, влияющие только на разблокировку мини‑игр), колесом жизни,
утренними/вечерними чек‑апами, месячной историей в виде тепловой карты,
числовым «psycho mode» и вводным онбордингом.

Дата выгрузки: 24 июня 2026.

---

## 1. Технологии и структура файлов

- **Чистый фронтенд без зависимостей и сборки.** Vanilla JS, HTML, CSS. Графики
  рисуются вручную на `<canvas>` (без Chart.js и т.п.). Открывается как статика.
- **Хранение — `localStorage`**, один ключ `habbittracker_progress` (JSON).
  Бэкенда нет, всё локально в браузере.

```
habbittracker-main/
├── index.html          — разметка всех экранов и модалок
├── habbittracker.css   — вся стилизация (дизайн‑система ч/б)
├── habbittracker.js    — вся логика (≈2150 строк, один IIFE на DOMContentLoaded)
├── pics/               — изображения (волки для питомца/интро, игральные карты)
├── PROJECT.md          — этот документ (пользовательская дока)
├── HANDOFF.md          — технический «map» для разработки
└── .claude/launch.json — конфиг локального превью (python http.server)
```

Запуск локально: `python3 -m http.server 4599` в папке проекта, затем открыть
`http://localhost:4599`.

---

## 2. Поток приложения (экраны)

```
Интро (волк + фраза)
        │  тап по экрану
        ▼
  Дашборд ──► тумблер «psycho mode» + нижний таб‑бар из 6 вкладок:
        ├── День      (привычки на сегодня + колесо жизни)   ← стартовая
        ├── Месяц     (история / тепловая карта)
        ├── Игры      (мини‑игры, часть закрыта)
        ├── Утро      (утренний чек‑ап)
        ├── Вечер     (вечерний чек‑ап)
        └── Питание   (дневник еды + недельный график)
```

- **Новый пользователь:** видит интро → тап → создаётся стартовый набор (3 привычки)
  → попадает на «День», где сразу запускается короткий обучающий тур (см. §10).
- **Вернувшийся пользователь:** интро пропускается, сразу «День» (тур не показывается).
- Экранов выбора идентичности/целей и «эволюции» **нет** (удалены).
- Есть 6‑я вкладка **«Питомец»**, но она пока **скрыта** (код готов, ждёт визуал — см. §11).

## 3. Блоки и экраны — детально

### 3.1. Интро
- Изображение волка (`pics/`, grayscale), плавающая анимация.
- Мотивирующая фраза, меняется каждые 5 секунд (по кругу).
- Подсказка «нажми на экран…». Тап → загрузочный оверлей → «День».

### 3.2. День (вкладка «День»)
Главный экран трекинга.
- **Шапка (общая для всех вкладок):** сверху ряд из двух круглых кнопок — слева сброс
  `↺`, справа «?» (повтор обучающего тура). Ниже — текущая дата и «УРОВЕНЬ N».
- **Заголовок** «Задачи на сегодня» + таймер «до обновления: ЧЧ:ММ:СС»
  (отсчёт до полуночи, когда отметки сбрасываются).
- **Список привычек (бинарная отметка):** строка = чёрно‑белый чекбокс + название +
  (серия) + «⋯».
  - Клик по чекбоксу или названию — отметить/снять выполнение за сегодня.
  - Выполненная привычка: залитый чёрный чекбокс с галочкой + зачёркивание.
  - Если есть серия — справа моно‑огонёк + число.
  - «⋯» открывает модалку настроек привычки.
- **Добавление:** строка с полем «+ добавить привычку», ввод + Enter. Лимит — **10
  привычек**. При 10/10 поле заменяется надписью «Максимум 10 привычек».
- **Колесо жизни** (под списком привычек, без заголовка) — см. §4.
- **Подвал (общий):** прогресс‑бар опыта до следующего уровня + «X / Y XP» и «%».

> В режиме **psycho mode** (тумблер над таб‑баром) «День» заменяется на числовые
> показатели — см. §7.

### 3.3. Месяц (вкладка «Месяц») — история / тепловая карта
- **Навигация по месяцам:** `← Июнь 2026 →`.
- **Сводка месяца:** «выполнено / возможно / прогресс %» + прогресс‑бар.
- **Тепловая карта (в стиле GitHub):** по строке на каждую привычку, по клетке на
  день месяца. Залитая клетка = выполнено, «сегодня» обведено рамкой, будущие дни
  заблокированы. Справа от строки — серия (моно‑огонёк) и счётчик `N/<дней в месяце>`.
  - **Редактируемая:** клик по клетке отмечает/снимает день задним числом.
  - **Опыт за сегодня:** отметка привычки за **сегодняшний** день начисляет XP так же,
    как на «Дне» (один раз в день, без фарма). Правка **прошлых** дней меняет лог и
    серии, но XP за неё не даётся.
- **Колесо жизни за месяц** (под картой).
- **График «Настроение и качество сна»:** линия настроения (сплошная) и качества
  сна (пунктир) за месяц — данные берутся из утренних чек‑апов.

> В **psycho mode** «Месяц» показывает сводку по числовым показателям (сумма за месяц
> против дневной цели × число дней), **без** графика настроения/сна.

### 3.4. Игры (вкладка «Игры»)
Четыре мини‑игры; открытость зависит от уровня (см. §6).
- Закрытые игры показаны пунктиром с замком и подписью, на каком уровне откроются.
- **Найди пару** — мемори: пары из колоды игральных карт (`pics/`). Открой все пары.
- **Посчитай** — устный счёт на время. XP = `max(1, min(10, верных))`.
- **10 слов** — запомнить набор слов за отведённое время, затем ввести.
- **Быстрое судоку** — сетка 9×9 с одним пропуском в каждом квадрате 3×3; заполни
  пропуски.

Все игры начисляют XP за результат.

### 3.5. Утро (вкладка «Утро») — утренний чек‑ап
Поля формы:
- Во сколько лёг в кровать? (время)
- Во сколько уснул? (время)
- Во сколько проснулся? (время)
- Качество сна (шкала 1–10)
- Уровень энергии утром (1–10)
- **Настроение** (1–10)
- Главный фокус на сегодня (текст)

Кнопки: «Сохранить чек‑ап», «Аналитика». После сохранения форма блокируется,
появляется «Редактировать». Иконка‑календарь в шапке — просмотр истории за другой
день. Первое сохранение за день даёт **+3 XP**.

### 3.6. Вечер (вкладка «Вечер») — вечерний чек‑ап
Поля формы:
- Общая оценка дня (1–10)
- Уровень энергии сейчас (1–10)
- Удовлетворённость результатами (1–10)
- Уровень спокойствия (1–10)
- Качество выполнения привычек (1–10)
- За что благодарен сегодня? (текст)
- Что улучшить завтра? (текст)

Поведение — как у утреннего (сохранение, блокировка, история, +3 XP).

### 3.7. Аналитика (модалка из чек‑апов)
По кнопке «Аналитика» открывается полноэкранная модалка: выбор диапазона дат и
линейные графики по каждому числовому показателю чек‑апа за период.

### 3.8. Модалки
- **Настройки привычки** (по «⋯»): редактирование названия, формула «Я сделаю это
  после того, как…» (триггер), переключатель «Напомнить в:» + время, выбор **сфер
  колеса жизни** (чипы) и кнопка **«Удалить»**.
- **Подтверждение удаления** — единая стилизованная ч/б модалка (а не системный
  диалог браузера) для удаления **привычки** и **показателя**: «Удалить … «X»?»
  с кнопками «Отмена» / «Удалить». «Отмена» возвращает к настройкам, Esc — закрыть.
- **Разблокировка игры** (на ур. 3 / 7 / 10): «Новая игра открыта — выбери, что
  добавить» со списком ещё закрытых игр. При самом первом заходе в «Игры» — выбор
  первой игры.

### 3.9. Напоминания
Периодически проверяется: если у привычки задано время напоминания и оно совпало с
текущим, а привычка не выполнена — всплывает тост «Время действовать» с названием
привычки.

### 3.10. Питание (вкладка «Питание»)
Простой дневник еды.
- **Сегодня:** три приёма — Завтрак, Обед, Ужин; у каждого поле времени и поле «что кушал».
  Записи сохраняются автоматически по мере ввода (без кнопки «Сохранить»).
- **Эта неделя:** недельный график (Пн–Вс). Для каждого дня — горизонтальная шкала времени
  (5:00–24:00) с точками приёмов пищи по их времени и подписями «время · что съел».
  Сегодняшний день подсвечён.

## 4. Колесо жизни

Под списком привычек (на «Дне» и в «Месяце») отображается «колесо жизни» — круговая
диаграмма из 7 сфер:

**Карьера · Дом · Энергия · Финансы · Социальная жизнь · Саморазвитие · Эмоциональное состояние.**

- Заполняется **автоматически**: в настройках привычки («⋯») можно привязать её к
  одной или нескольким сферам; по мере выполнения таких привычек соответствующие
  секторы колеса заполняются. Это не ручная самооценка, а отражение фактических
  действий.
- Пока ни одна привычка не привязана к сферам — вместо колеса показывается подсказка,
  как его заполнить.

1. Подъём до 6 утра
2. Книга
3. Тренировка

## 6. Серии (стрики)

- Серия привычки = число подряд идущих выполненных дней вплоть до сегодня.
- Если сегодня ещё не отмечено — серия **не рвётся**, отсчёт идёт со вчера.
- Считается из постоянного лога (`history`), показывается на «Дне» и в «Месяце».

## 7. Опыт, уровни и разблокировка игр

**Важно:** опыт и уровни сейчас влияют **только** на количество открытых мини‑игр
(и на стадию питомца, который пока скрыт). На привычки, лимиты и прочее не влияют.

### Что даёт XP
| Действие | XP |
|---|---|
| Выполнить привычку (на «Дне» или отметив сегодняшний день в «Месяце») | `5 + (уровень − 1) × 3` (ур.1 → 5, ур.2 → 8, ур.3 → 11…) |
| Сохранить чек‑ап (первый раз за день) | +3 |
| Мини‑игры | за результат |

Защита от «фарма»: за одну привычку XP начисляется **не больше одного раза в день**
(повторное снятие/постановка галочки опыт не добавляет; счётчик общий между «Днём»
и «Месяцем»).

### Уровни
- Нужно XP до следующего уровня: `floor(15 × уровень^1.8)`
  (ур.1 → 15, ур.2 → 52, ур.3 → 108, далее растёт).
- При достижении порога: уровень +1, текущий XP обнуляется (без переноса остатка).

### Разблокировка игр
- **Первую** игру игрок выбирает сам при первом заходе в «Игры».
- На **ур. 3**, **ур. 7** и **ур. 10** при повышении уровня всплывает выбор: какую из
  ещё закрытых игр открыть. Всего игр — 4.

## 8. Psycho mode (числовые показатели)

Тумблер «psycho mode» над таб‑баром переключает «День» с бинарных привычек на
**числовые показатели дня** — отдельный список, не связанный с привычками.

- **Показатели по умолчанию (7):** км пробежал (цель 10 км), часов поспал (цель 8 ч),
  денег заработал (цель 3000 ₽), минут медитировал (цель 15 мин), страниц прочитал
  (цель 30 стр), сигарет скурил (лимит 0 шт), кофе выпил (лимит 2 чашек).
- **Тип показателя:** «цель» (больше = лучше) или «лимит» (меньше = лучше). У лимита —
  бейдж «лимит» и пометка «превышен» при выходе за рамки.
- **Ввод за день:** поле «+ значение» добавляет число к накопленному за сегодня;
  прогресс‑бар показывает выполнение относительно цели/лимита. Есть кнопка «сброс».
- **Цель/лимит редактируется** инлайн (кнопка «цель …»/«лимит …»).
- **Добавление/удаление:** внизу — «+ добавить показатель» (название, тип цель/лимит,
  значение, единица). У каждого показателя — «удалить» (через стилизованное
  подтверждение). Список можно опустошить полностью.
- **Инверсия цветов:** в psycho mode весь дашборд инвертируется (чёрный фон) —
  визуальный «режим фокуса».

## 9. Модель данных (localStorage)

Ключ: `habbittracker_progress`. Один JSON‑объект:

```js
{
  level: 1,
  currentXP: 0,
  habits: [
    {
      uid: "u…",           // стабильный уникальный id привычки
      text: "Книга",
      completed: false,     // отметка за СЕГОДНЯ (сбрасывается на новый день)
      xpDate: "2026-06-24", // дата, когда за неё уже начислили XP (анти‑фарм)
      areas: ["growth"],    // привязка к сферам колеса жизни
      triggerText: "",      // «после того как…» (опц.)
      reminderTime: null    // "08:00" или null (опц.)
    }
  ],
  unlockedGames: ["memory"],            // открытые игры
  lastActiveDate: "2026-06-24",
  checkins: { morning: {}, evening: {} },          // черновик чек‑апов на сегодня
  checkinHistory: {                                // сохранённые чек‑апы по дням
    "2026-06-24": { morning: {…}, evening: {…} }
  },
  history: {                                       // постоянный лог выполнения
    "2026-06-24": { "u…": true }                   // дата → { uid: true }
  },
  foodLog: {                                       // дневник еды по дням
    "2026-06-24": { breakfast: { time: "08:00", text: "овсянка" } }
  },
  psychoMode: false,                              // включён ли числовой режим
  metrics: [                                      // живой список показателей (psycho)
    { id: "…", name: "км пробежал", type: "goal", target: 10, unit: "км" }
  ],
  metricTargets: { "…": 12 },                     // переопределённые дневные цели
  metricLog: {                                    // числовые значения по дням
    "2026-06-24": { "…": 5 }
  },
  onboardingDone: false,                          // пройден ли вводный тур
  seenHints: { month: true }                      // показанные контекстные подсказки
}
```

- `history` — это «вечная память» трекера: по нему строятся тепловая карта и серии.
  Не очищается при смене дня.
- Смена дня (`checkNewDay`): на новую календарную дату обнуляются `completed` у
  привычек и черновик `checkins`; `history`, `checkinHistory` и `metricLog` сохраняются.

## 10. Онбординг

- **Обучающий тур** по «Дню» — короткие подсказки с затемнением и подсветкой
  элементов (чекбокс, «⋯», добавление, колесо жизни, psycho‑тумблер, таб‑бар).
  Показывается новому пользователю после интро и повторяется по кнопке «?». Кнопки
  «Пропустить» / «Далее» / «Готово».
- **Контекстные подсказки** при первом заходе в «Месяц», «Утро», «Вечер» — короткий
  баннер с пояснением и кнопкой «Понятно» (показывается один раз на вкладку).

## 11. Питомец (пока скрыт)

В коде есть вкладка «Питомец» (волк, развивающийся с уровнем игрока: Щенок → Подросток
→ Взрослый → Вожак; настроение зависит от «заботы» — выполнения привычек и чек‑апов за
неделю). Вкладка **временно скрыта** и ждёт полный набор изображений. Когда ассеты
будут готовы — вкладку легко вернуть (см. `HANDOFF.md`).

## 12. Что можно делать на сайте (интеракции)

- Пройти интро тапом и получить стартовый набор привычек + обучающий тур.
- Отмечать/снимать выполнение привычек за сегодня (чекбокс или название).
- Добавлять привычки (до 10), переименовывать, задавать триггер, время напоминания,
  привязывать к сферам колеса жизни, удалять (со стилизованным подтверждением).
- Видеть, как колесо жизни заполняется от выполнения привычек.
- Получать опыт и повышать уровень за привычки (на «Дне» и за сегодняшние дни в
  «Месяце»), чек‑апы и игры.
- Выбирать первую мини‑игру и открывать новые на ур. 3 / 7 / 10; играть и зарабатывать XP.
- Заполнять утренний и вечерний чек‑апы; редактировать их; смотреть за прошлые дни.
- Вести дневник питания (завтрак/обед/ужин: время + что съел) и смотреть недельный график по времени.
- Открывать аналитику чек‑апов (графики по диапазону дат).
- Смотреть месячную тепловую карту, листать месяцы, отмечать дни задним числом,
  видеть серии и график настроения/сна.
- Включать **psycho mode**: вести числовые показатели дня, добавлять/удалять их,
  задавать цели/лимиты; смотреть месячную сводку по показателям.
- Получать всплывающие напоминания по привычкам в заданное время.
- Сбросить весь прогресс кнопкой `↺` и перепройти тур кнопкой «?».

## 13. Дизайн‑система

- **Палитра — чёрно‑белая.** Основные цвета: `#111` (чёрный), `#fff` (белый),
  оттенки серого. Цветных акцентов нет (серии‑огоньки тоже монохромные). В psycho
  mode весь дашборд инвертируется.
- **Шрифт:** Inter / system‑ui.
- **Иконки:** тонкие монохромные SVG (таб‑бар, календарь, «троеточие», замок,
  огонёк). Эмодзи в интерфейсе убраны.
- **Адаптив:** одноколоночная мобильная вёрстка; таб‑бар из 6 видимых вкладок
  (иконка + подпись), активная вкладка — чёрная.

## 14. Что можно доработать дальше (идеи и мелочи)

**Идеи развития**
- Вернуть питомца, когда будут готовы изображения всех стадий/настроений; добавить
  idle‑анимацию и празднование смены стадии/уровня.
- Экспорт/импорт данных (JSON), синхронизация между устройствами (нужен бэкенд).
- Напоминания через настоящие Web Notifications (сейчас — только тост в открытой
  вкладке, и только пока сайт открыт).
- Гибкая периодичность привычек (не только «каждый день»).
- Перенос несгораемого остатка XP при повышении уровня (сейчас обнуляется).

**Мелочи/тех‑долг**
- Тост напоминания всё ещё содержит эмодзи 🔔 — можно заменить на моно‑иконку для
  полного соответствия ч/б стилю.
- В `pics/` есть неиспользуемые ассеты (`Sneaky wolf 1–6`, `wolf_master`) — можно
  почистить, когда определится финальный визуал питомца/интро.
- Дата «сегодня» в логах считается по локальному времени, а часть старой логики
  чек‑апов — по UTC; на стыке полуночи в редких часовых поясах возможно расхождение
  в один день (данные при этом не теряются).
